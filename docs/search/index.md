# Search

## Overview

This project uses [Typesense](https://typesense.org/) as its full-text search backend. The Typesense container ships with the default Sail stack and starts automatically alongside the other services. Laravel Scout — the framework-level package that bridges Eloquent models to a search driver — is **not yet installed**, so no model data flows to the Typesense index today.

This page documents the infrastructure that is in place, the application-side gap, and how to verify the service locally.

## Infrastructure in Place

### Typesense Docker service

The `typesense` service is declared in `compose.yaml`:

```yaml
# compose.yaml
typesense:
    image: 'typesense/typesense:27.1'
    ports:
        - '${FORWARD_TYPESENSE_PORT:-8108}:8108'
    environment:
        TYPESENSE_DATA_DIR: '${TYPESENSE_DATA_DIR:-/typesense-data}'
        TYPESENSE_API_KEY: '${TYPESENSE_API_KEY:-xyz}'
        TYPESENSE_ENABLE_CORS: '${TYPESENSE_ENABLE_CORS:-true}'
    volumes:
        - 'sail-typesense:/typesense-data'
```

Key details:

| Property | Value |
| -------- | ----- |
| Image | `typesense/typesense:27.1` |
| Internal port | `8108` |
| Host port (default) | `8108` (configurable via `FORWARD_TYPESENSE_PORT`) |
| Data volume | `sail-typesense` (named Docker volume, persists across container restarts) |
| CORS | Enabled by default (`TYPESENSE_ENABLE_CORS=true`) |

A healthcheck is also defined: Sail polls `GET /health HTTP/1.1` on port 8108 with five retries and a 7-second timeout before marking the service healthy.

### Environment variables

`.env.example` declares all five Scout/Typesense keys:

```dotenv
SCOUT_DRIVER=typesense
TYPESENSE_HOST=typesense
TYPESENSE_PORT=8108
TYPESENSE_PROTOCOL=http
TYPESENSE_API_KEY=xyz
```

These same values are active in the default `.env` for local development. `TYPESENSE_HOST=typesense` resolves to the container name inside the Sail Docker network, which is the correct address for service-to-service communication.

## Application-Side Gap

The following checks were run against `HEAD` to document what is not yet wired.

### No Laravel Scout package

`composer.json` does not declare `laravel/scout`. Running `composer show | grep scout` inside the container returns nothing. Without this package, Laravel has no Scout service provider, no `Artisan scout:*` commands, and no driver resolution for Typesense.

### No Scout configuration file

`config/scout.php` does not exist. The file is generated by `artisan vendor:publish --provider="Laravel\Scout\ScoutServiceProvider"` after Scout is installed. Until then, no Scout configuration is available in the application.

### No Searchable models

A search of `app/` finds no class that uses the `Laravel\Scout\Searchable` trait. No Eloquent model is currently set up to push data to — or pull results from — Typesense.

## Verifying the Service Locally

Even without Scout wired up, you can confirm the Typesense container responds correctly once Sail is running:

```bash
# From the host machine — Sail forwards container port 8108 to localhost:8108
curl http://localhost:8108/health
```

A healthy response looks like:

```json
{"ok":true}
```

You can also list collections (will be empty until Scout creates them):

```bash
curl -H "X-TYPESENSE-API-KEY: xyz" http://localhost:8108/collections
```

Replace `xyz` with the value of `TYPESENSE_API_KEY` in your `.env` if you changed the default.

## Not Yet Implemented

::: warning Scout package installation is deferred

`laravel/scout` is not installed. The Typesense container runs and is reachable, but no application data is indexed. Eloquent models do not implement `Searchable`, and no `config/scout.php` configuration file exists.

A future milestone will install Scout, publish its config, and wire the Typesense driver to one or more models.

:::
